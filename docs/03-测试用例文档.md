# PostgreSQL AOT 数据库连接库测试用例文档

日期：2026-05-24

## 1. 测试目标

本测试文档是后续编写测试代码的规格说明。每个功能点必须至少覆盖：

- SQL 翻译是否正确。
- 参数顺序、参数类型和值是否正确。
- PostgreSQL 实际执行结果是否正确。
- NativeAOT 发布后是否仍可运行。
- 不支持场景是否抛出明确异常。
- 是否存在 SQL 注入、客户端执行、反射热路径或性能退化风险。

## 2. 测试项目划分

建议测试项目：

```text
tests/
  Perigon.PostgreSQL.Tests/
    SQL translation snapshots
    metadata tests
    analyzer tests
  Perigon.PostgreSQL.IntegrationTests/
    PostgreSQL real database tests
  Perigon.PostgreSQL.NativeAotSmokeTests/
    publish and run native binary
  Perigon.PostgreSQL.Benchmarks/
    BenchmarkDotNet or custom benchmark harness
```

## 3. 测试数据库模型

### 3.1 User

字段：

- `id int primary key generated always as identity`
- `user_name text not null`
- `age int not null`
- `status text null`
- `group_name text null`
- `tags text[] null`
- `scores int[] null`
- `profile jsonb null`
- `created_at timestamptz not null default now()`
- `updated_at timestamptz null`
- `version int not null default 1`

### 3.2 Blog

字段：

- `id int primary key generated always as identity`
- `user_id int not null references users(id)`
- `name text not null`
- `tags text[] null`
- `metadata jsonb null`
- `created_at timestamptz not null default now()`

### 3.3 Post

字段：

- `id int primary key generated always as identity`
- `blog_id int not null references blogs(id)`
- `author_id int not null references users(id)`
- `title text not null`
- `content text null`
- `published bool not null`
- `rating numeric null`
- `created_at timestamptz not null default now()`

### 3.4 Comment

字段：

- `id int primary key generated always as identity`
- `post_id int not null references posts(id)`
- `author_id int not null references users(id)`
- `content text not null`
- `created_at timestamptz not null default now()`

### 3.5 PostTag

字段：

- `post_id int not null references posts(id)`
- `tag text not null`
- primary key `(post_id, tag)`

## 4. Seed 数据

必须提供固定 seed：

- 10 users：用于基础查询。
- 100 users：用于分页、排序、include。
- 10000 users：用于性能和批量场景。
- 用户无 blog。
- 用户 1 个 blog。
- 用户多个 blog。
- blog 无 post。
- blog 多 post。
- post 无 comment。
- post 多 comment。
- `tags = NULL`。
- `tags = []`。
- `tags = ['developer', 'postgres']`。
- `scores = NULL`。
- `scores = []`。
- `scores = [1, 2, 3]`。
- `profile = NULL`。
- `profile = '{}'`。
- `profile = '{"name":"Alice","level":3}'`。
- 包含单引号、双引号、反斜杠、换行、emoji 的文本。

## 27. 测试完成定义

一个功能只有同时满足以下条件，才视为完成：

- 有 SQL 翻译快照。
- 有真实 PostgreSQL 集成测试。
- 有 NULL/空集合/边界值测试。
- 有不支持场景测试。
- 若涉及 AOT，必须通过 NativeAOT smoke test。
- 若涉及写入，必须覆盖事务回滚。
- 若涉及批量，必须覆盖 0 行、1 行、大批量。
- 若涉及 SQL 生成，必须覆盖 SQL 注入输入。
