# PostgreSQL AOT 数据库连接库设计文档

日期：2026-05-24

## 1. 项目定位

本项目目标是开发一个专用于 PostgreSQL 的 .NET 数据库连接库。它提供接近 EF Core 的 DbContext/DbSet 使用体验，同时以 NativeAOT、无追踪、高性能批量写入和 PostgreSQL 原生能力为核心设计目标。

该库不是通用 ORM，也不是完整 EF Core 替代品。它面向以下场景：

- 需要 .NET NativeAOT 发布的应用。
- 需要较快启动速度、较低运行时开销和较小部署体积的服务。
- 使用 PostgreSQL 作为唯一数据库，不需要跨数据库 provider 抽象。
- 希望使用类似 EF Core 的 LINQ 查询风格，但不需要变更追踪和 `SaveChanges`。
- 需要原生支持 `COPY`、`ON CONFLICT`、数组、JSONB、`RETURNING`、filtered include、聚合查询等 PostgreSQL 能力。

## 2. 设计目标

### 2.1 AOT 优先

库的所有核心路径都必须兼容 NativeAOT 和 trimming：

- 不依赖运行时实体扫描。
- 不依赖 `Expression.Compile()` 作为执行路径。
- 不依赖 `Reflection.Emit`。
- 不在热路径使用 `PropertyInfo.GetValue/SetValue`。
- 通过源生成器生成实体元数据、materializer 和批量写入访问器。

### 2.2 PostgreSQL 专用

项目明确只支持 PostgreSQL，因此可以直接暴露和优化 PostgreSQL 特性：

- PostgreSQL 原生位置参数：`$1`, `$2`。
- Binary `COPY FROM STDIN` 批量插入。
- `INSERT ... RETURNING`。
- `INSERT ... ON CONFLICT` upsert。
- PostgreSQL 15+ `MERGE`。
- 数组类型与数组运算符。
- JSONB 运算符。
- schema-qualified table、大小写敏感标识符和保留字引用。

### 2.3 无追踪

库默认不做变更追踪，不提供 EF Core 式 `SaveChanges` 工作流。

写入行为以立即执行为主：

- `InsertAsync`
- `ExecuteUpdateAsync`
- `ExecuteDeleteAsync`
- `BulkInsertAsync`
- `InsertManyReturningAsync`
- `UpsertManyAsync`
- `MergeAsync`

这种设计减少运行时状态、内存占用和 AOT 复杂度，也让写入行为更明确。

### 2.4 类 EF Core 使用体验

用户应能使用熟悉的 DbContext/DbSet 查询方式：

```csharp
var users = await db.Users
    .Where(u => u.Age > 18)
    .OrderBy(u => u.Name)
    .ToListAsync();

await db.Users
    .Where(u => u.Age < 18)
    .ExecuteDeleteAsync();
```

但库不承诺完整兼容 EF Core 的所有 LINQ 语义。所有支持范围必须文档化，并通过测试锁定。

## 3. 目标用户

### 3.1 应用开发者

希望以较少样板代码访问 PostgreSQL，同时获得 AOT、批量写入和类型安全查询。

### 3.2 服务端系统开发者

构建高启动速度、低内存占用的 HTTP API、后台服务、CLI 工具、边缘服务或 serverless 工作负载。

### 3.3 性能敏感场景

需要快速批量导入、批量更新、批量删除，以及可预测 SQL 的业务系统。

## 4. 非目标

本项目不追求以下能力：

- 跨数据库兼容。
- EF Core 完整兼容。
- 变更追踪。
- lazy loading。
- 动态代理。
- 自动迁移完整替代品。
- 动态 JSON POCO 映射。
- 未声明支持的 LINQ 表达式自动回退到客户端执行。

## 5. 核心能力范围

### 5.1 查询

基础查询能力：

- `Where`
- `OrderBy`
- `ThenBy`
- `Skip`
- `Take`
- `Select`
- `FirstOrDefaultAsync`
- `SingleOrDefaultAsync`
- `ToListAsync`
- `CountAsync`
- `AnyAsync`

进阶查询能力：

- filtered include
- split query include
- single query include
- GroupBy 聚合投影
- 显式 Join
- 原生 SQL 查询

### 5.2 写入

单条写入：

- `InsertAsync(entity)`
- `InsertAsync(entity, returning: false)`

条件 DML：

- `ExecuteUpdateAsync`
- `ExecuteDeleteAsync`

批量写入：

- `BulkInsertAsync`，默认使用 binary COPY。
- `InsertManyReturningAsync`，使用多值 `INSERT ... RETURNING`。
- `UpsertManyAsync`，使用 `INSERT ... ON CONFLICT`。
- `MergeAsync`，仅 PostgreSQL 15+。

### 5.3 类型支持

基础类型：

- 整数、浮点、decimal、bool。
- string。
- Guid。
- DateTime、DateTimeOffset、DateOnly、TimeOnly。
- byte array。

PostgreSQL 特有类型：

- 数组：`T[]`、`List<T>`。
- JSONB：`string`、`JsonDocument`、`JsonElement`、源生成 JSON context。
- UUID。
- inet/cidr 等后续可扩展类型。

动态 JSON POCO 不支持。

## 6. 高层架构

```text
用户代码
  |
  v
DbContext / DbSet
  |
  v
Query Provider / Query Model
  |
  v
Expression Translator
  |
  v
SQL Builder
  |
  v
Command Executor
  |
  v
NpgsqlDataSource / NpgsqlConnection
  |
  v
PostgreSQL
```

辅助组件：

- Source Generator：生成实体模型、materializer、属性 getter、DbContext glue code。
- Analyzer：提示不支持表达式、AOT 不安全用法、未映射实体。
- Benchmark：验证性能基线和回归。
- NativeAOT Smoke Tests：验证发布后的原生二进制。

## 7. 典型使用方式

### 7.1 实体定义

```csharp
[Table("users")]
public sealed class User
{
    [Column("id", IsPrimaryKey = true)]
    public int Id { get; set; }

    [Column("user_name")]
    public string Name { get; set; } = "";

    [Column("age")]
    public int Age { get; set; }

    [Column("tags")]
    public string[] Tags { get; set; } = [];

    [Column("profile", TypeName = "jsonb")]
    public string ProfileJson { get; set; } = "{}";
}
```

### 7.2 DbContext

```csharp
public sealed class AppDbContext : PostgresDbContext
{
    public DbSet<User> Users => Set<User>();
    public DbSet<Blog> Blogs => Set<Blog>();

    protected override void OnConfiguring(PostgresDbContextOptionsBuilder builder)
    {
        builder.UsePostgres(connectionString);
    }
}
```

### 7.3 查询

```csharp
var users = await db.Users
    .Where(u => u.Age > 18 && u.Tags.Contains("developer"))
    .OrderBy(u => u.Name)
    .ToListAsync();
```

### 7.4 单条插入

```csharp
var inserted = await db.Users.InsertAsync(new User
{
    Name = "Alice",
    Age = 30,
    Tags = ["developer", "postgres"]
});
```

### 7.5 条件更新

```csharp
await db.Users
    .Where(u => u.Age > 60)
    .ExecuteUpdateAsync(s => s
        .Set(u => u.Status, "retired"));
```

### 7.6 条件删除

```csharp
await db.Users
    .Where(u => u.Age < 18)
    .ExecuteDeleteAsync();
```

### 7.7 批量插入

```csharp
await db.Users.BulkInsertAsync(users);
```

### 7.8 关联加载

```csharp
var users = await db.Users
    .Where(u => u.Group == "admin")
    .Include(u => u.Blogs.Where(b => b.Name == "postgres"))
    .ToListAsync();
```

## 8. 设计边界

### 8.1 不支持动态 JSON POCO

强类型 JSON 必须使用明确的序列化上下文或由用户自行处理为 `string`/`JsonDocument`。

### 8.2 不做静默客户端执行

任何无法翻译到 SQL 的表达式都必须抛出明确异常，不能自动转为内存执行。

### 8.3 不做隐式全表写入

无条件 `UPDATE` 和 `DELETE` 必须显式开启选项。

### 8.4 Include 和 GroupBy 需要定义结果形状

filtered include、split query、single query、GroupBy 聚合投影都可以支持，但必须为每种形态定义 SQL 输出、对象 materialization 和异常边界。

## 9. 成功标准

项目达到可用状态需要满足：

- NativeAOT publish 无警告或仅有明确可解释警告。
- 发布后的原生二进制可执行完整 CRUD。
- 所有 SQL 翻译均有快照测试。
- 所有数据库行为均有真实 PostgreSQL 集成测试。
- COPY 不退化为逐条 INSERT。
- materializer 热路径不使用反射。
- 不支持表达式有明确错误消息。
- CI 包含 PostgreSQL matrix 和 AOT smoke test。
