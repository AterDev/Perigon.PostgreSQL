# PostgreSQL AOT 数据库连接库详细技术文档

日期：2026-05-24

## 1. 技术栈

目标技术栈：

- .NET 10
- NativeAOT
- Npgsql 10.0
- PostgreSQL 15+
- C# Source Generator
- Roslyn Analyzer
- Microsoft.Extensions.DependencyInjection
- Microsoft.Extensions.Logging.Abstractions
- System.Linq.Expressions

PostgreSQL 版本建议：

- PostgreSQL 15 是最低建议版本，因为 `MERGE` 从 PostgreSQL 15 开始支持。
- `ON CONFLICT` 支持 PostgreSQL 9.5+，但本库建议整体测试矩阵从 PostgreSQL 15 开始。
- PostgreSQL 18 可作为前沿版本测试目标。

## 2. AOT 约束

### 2.1 必须避免

- `Expression.Compile()` 作为查询执行路径。
- `Activator.CreateInstance` 创建实体。
- 热路径 `PropertyInfo.GetValue` 和 `PropertyInfo.SetValue`。
- 运行时扫描所有程序集查找实体。
- 动态代理。
- lazy loading。
- `Reflection.Emit`。
- 动态 JSON POCO 映射。

### 2.2 必须启用

库项目建议配置：

```xml
<PropertyGroup>
  <TargetFramework>net10.0</TargetFramework>
  <IsAotCompatible>true</IsAotCompatible>
  <EnableTrimAnalyzer>true</EnableTrimAnalyzer>
  <EnableAotAnalyzer>true</EnableAotAnalyzer>
  <VerifyReferenceAotCompatibility>true</VerifyReferenceAotCompatibility>
</PropertyGroup>
```

### 2.3 源生成优先

以下内容应由源生成器生成：

- 实体到表映射。
- 属性到列映射。
- 主键、identity、generated、default column 元数据。
- `NpgsqlDataReader` 到实体的 materializer。
- 实体属性 getter，用于 INSERT/COPY。
- DbContext `Set<T>()` 注册和实体模型索引。

## 4. 核心组件

### 4.1 DbContext

职责：

- 持有配置。
- 创建 DbSet。
- 管理 `NpgsqlDataSource`。
- 管理事务作用域。
- 暴露原生 SQL API。

建议 API：

```csharp
public abstract class DbContext : IAsyncDisposable
{
    protected DbSet<T> Set<T>() where T : class;
    protected virtual void OnConfiguring(DbContextOptionsBuilder builder) { }
    protected virtual void OnModelCreating(ModelBuilder modelBuilder) { }

    public Task<TResult> TransactionAsync<TResult>(
        Func<CancellationToken, Task<TResult>> action,
        CancellationToken cancellationToken = default);

    public RawSqlQuery<T> SqlQuery<T>(FormattableString sql);
    public RawSqlCommand SqlCommand(FormattableString sql);
}
```

### 4.2 DbSet

职责：

- 实现查询入口。
- 持有实体元数据。
- 提供写入 API。
- 创建 query model。

建议 API：

```csharp
public sealed class DbSet<T> : IQueryable<T> where T : class
{
    public Task<List<T>> ToListAsync(CancellationToken cancellationToken = default);
    public Task<T?> FirstOrDefaultAsync(CancellationToken cancellationToken = default);
    public Task<T?> SingleOrDefaultAsync(CancellationToken cancellationToken = default);
    public Task<int> CountAsync(CancellationToken cancellationToken = default);
    public Task<bool> AnyAsync(CancellationToken cancellationToken = default);

    public Task<T> InsertAsync(T entity, CancellationToken cancellationToken = default);
    public Task<int> InsertAsync(T entity, bool returning, CancellationToken cancellationToken = default);

    public Task<int> ExecuteDeleteAsync(CancellationToken cancellationToken = default);
    public Task<int> ExecuteUpdateAsync(
        Expression<Func<UpdateSetters<T>, UpdateSetters<T>>> setters,
        CancellationToken cancellationToken = default);

    public Task BulkInsertAsync(
        IEnumerable<T> entities,
        BulkInsertOptions? options = null,
        CancellationToken cancellationToken = default);

    public Task<List<T>> InsertManyReturningAsync(
        IEnumerable<T> entities,
        ReturningOptions<T>? options = null,
        CancellationToken cancellationToken = default);

    public Task<int> UpsertManyAsync(
        IEnumerable<T> entities,
        Expression<Func<T, object>> conflictKey,
        UpsertOptions<T>? options = null,
        CancellationToken cancellationToken = default);
}
```

## 5. 查询管线

### 5.1 总流程

```text
IQueryable expression
  -> QueryModel
  -> SQL AST / SQL Builder
  -> SQL text + parameter list
  -> NpgsqlCommand
  -> NpgsqlDataReader
  -> generated materializer
```

### 5.2 QueryModel

QueryModel 应保存结构化查询信息，而不是直接拼接字符串：

- Root entity。
- Predicate list。
- Ordering list。
- Projection。
- Skip。
- Take。
- Include tree。
- GroupBy keys。
- Aggregates。
- Query execution mode。

### 5.3 表达式翻译

支持表达式：

- 二元表达式：`==`, `!=`, `>`, `>=`, `<`, `<=`。
- 布尔组合：`&&`, `||`, `!`。
- null 比较。
- 成员访问。
- 捕获变量。
- nullable 成员。
- string 方法。
- array/list 方法。
- JSONB 扩展方法。
- GroupBy 聚合表达式。

不支持表达式必须抛出 `UnsupportedQueryExpressionException`。

异常消息应包含：

- 不支持的表达式片段。
- 实体类型。
- 成员名。
- 建议替代写法。

## 6. SQL 生成

### 6.1 参数

统一使用 PostgreSQL 原生位置参数：

```sql
WHERE user_name = $1 AND age > $2
```

参数顺序必须稳定，由 SQL 生成器统一分配。

### 6.2 标识符引用

所有 table、schema、column 必须通过 identifier quoter：

```sql
"public"."users"."user_name"
```

禁止用户输入直接拼接为标识符。

### 6.3 SELECT

基础 SQL：

```sql
SELECT u.id, u.user_name, u.age
FROM users AS u
WHERE u.age > $1
ORDER BY u.user_name ASC
LIMIT $2 OFFSET $3;
```

### 6.4 INSERT

默认返回生成值：

```sql
INSERT INTO users (user_name, age, tags)
VALUES ($1, $2, $3)
RETURNING id, user_name, age, tags;
```

不返回：

```sql
INSERT INTO users (user_name, age, tags)
VALUES ($1, $2, $3);
```

### 6.5 UPDATE

```sql
UPDATE users AS u
SET status = $1, updated_at = $2
WHERE u.age > $3;
```

表达式更新：

```sql
UPDATE users AS u
SET age = u.age + $1
WHERE u.id = $2;
```

### 6.6 DELETE

```sql
DELETE FROM users AS u
WHERE u.age < $1;
```

无条件 DELETE 默认禁止，除非显式配置。

### 6.7 COPY

使用 Npgsql binary import：

```sql
COPY users (user_name, age, tags)
FROM STDIN (FORMAT BINARY);
```

写入逻辑由源生成 getter 提供。

### 6.8 多值 INSERT RETURNING

```sql
INSERT INTO users (user_name, age)
VALUES ($1, $2), ($3, $4)
RETURNING id, user_name, age;
```

适合需要返回生成值的小到中批量写入。

### 6.9 Upsert

```sql
INSERT INTO users (id, user_name, age)
VALUES ($1, $2, $3)
ON CONFLICT (id)
DO UPDATE SET
  user_name = EXCLUDED.user_name,
  age = EXCLUDED.age
RETURNING id, user_name, age;
```

### 6.10 MERGE

仅 PostgreSQL 15+：

```sql
MERGE INTO users AS target
USING (VALUES ($1, $2)) AS source(id, user_name)
ON target.id = source.id
WHEN MATCHED THEN
  UPDATE SET user_name = source.user_name
WHEN NOT MATCHED THEN
  INSERT (id, user_name) VALUES (source.id, source.user_name);
```

## 7. 数组支持

### 7.1 Npgsql 层

Npgsql 原生支持 PostgreSQL array：

- `T[]`
- `List<T>`
- `NpgsqlDbType.Array | NpgsqlDbType.Integer`
- `NpgsqlDbType.Array | NpgsqlDbType.Text`

### 7.2 ORM 翻译层

建议翻译：

| C# | SQL |
| --- | --- |
| `u.Tags.Contains("developer")` | `u.tags @> ARRAY[$1]` |
| `u.Tags.Any(t => t == tag)` | `$1 = ANY(u.tags)` |
| `ids.Contains(u.Id)` | `u.id = ANY($1)` |
| `u.Tags.ContainsAll(tags)` | `u.tags @> $1` |
| `u.Tags.Overlaps(tags)` | `u.tags && $1` |
| `u.Tags.Length` | `cardinality(u.tags)` |

数组列查询应优先生成 GIN index 友好的 SQL。

## 8. JSONB 支持

### 8.1 支持范围

支持：

- `string` 映射 `jsonb`。
- `JsonDocument`。
- `JsonElement`。
- source-generated `JsonSerializerContext`。
- JSONB containment。
- key exists。
- path extract。

不支持：

- 动态 JSON POCO 映射。
- 无源生成上下文的任意对象图映射。

### 8.2 SQL 示例

```sql
WHERE profile @> $1
WHERE profile ? $1
WHERE profile ->> 'name' = $1
```

## 9. Include 技术方案

### 9.1 Split Query

默认方案：

```sql
SELECT u.*
FROM users AS u
WHERE u.group = $1;

SELECT b.*
FROM blogs AS b
WHERE b.user_id = ANY($2)
  AND b.name = $3;
```

优点：

- 避免父行重复。
- 避免多 collection include 的笛卡尔膨胀。
- materialization 简单。
- 更适合无追踪库。

### 9.2 Single Query

可选方案：

```sql
SELECT u.*, b.*
FROM users AS u
LEFT JOIN blogs AS b
  ON b.user_id = u.id
 AND b.name = $2
WHERE u.group = $1;
```

需要处理：

- 父实体去重。
- 子集合组装。
- 多 include 行膨胀。
- 分页改写。

### 9.3 Include 结果规则

- 父实体存在但子集合为空时，父实体必须返回。
- filtered include 只影响子集合，不应过滤父集合，除非用户在父查询中显式使用 `Where`。
- `Skip/Take` 应先作用于父查询，再加载子集合。

## 10. GroupBy 技术方案

### 10.1 支持 SQL 聚合投影

```csharp
db.Users
  .GroupBy(u => u.Group)
  .Select(g => new
  {
      Group = g.Key,
      Count = g.Count(),
      MaxAge = g.Max(u => u.Age)
  });
```

SQL：

```sql
SELECT u.group, count(*), max(u.age)
FROM users AS u
GROUP BY u.group;
```

### 10.2 不默认支持完整 IGrouping

以下形式不作为第一优先级：

```csharp
db.Users.GroupBy(u => u.Group).ToListAsync();
```

原因是 SQL 聚合结果和 LINQ `IGrouping<TKey,TElement>` 不是同一结果形状。

## 11. 源生成器

### 11.1 EntityModelGenerator

当前已实现基础版：扫描引用源生成器的项目中的 `DbSet<T>` 属性，为发现的实体生成 module initializer，并注册 `EntityModelRegistry`。运行时 `EntityModel.For<T>()` 会优先使用生成元数据，未注册类型再回退到反射建模。

生成：

- 表名。
- schema。
- 列列表。
- 主键。
- 写入列。
- 返回列。
- 导航关系（计划支持）。

### 11.2 MaterializerGenerator

当前已实现基础版：`EntityModelGenerator` 同时为发现的实体和可构造 DTO/结果类型生成 `NpgsqlDataReader -> T` materializer，并通过 module initializer 注册到 `EntityMaterializerRegistry`。执行查询时 `CommandExecutor.ExecuteQueryAsync<T>()` 优先使用生成 materializer；未注册类型继续使用反射 fallback。生成 materializer 会跳过缺失列和 `NULL` 列，以保持 partial `RETURNING`/raw SQL 兼容行为。

生成：

```csharp
public static User Materialize(NpgsqlDataReader reader)
{
    return new User
    {
        Id = reader.GetInt32(0),
        Name = reader.GetString(1),
        Tags = reader.GetFieldValue<string[]>(2)
    };
}
```

### 11.3 BulkWriterGenerator

当前已实现基础版：`EntityModelGenerator` 会为发现的实体生成属性 getter，并注册到 `EntityValueAccessorRegistry`。`INSERT`、`InsertManyReturning`、批量 `INSERT ... VALUES`、`UpsertMany` 和 Binary `COPY` 写入路径优先使用生成 getter 读取实体值；未注册类型继续使用反射 fallback。完整的 typed `NpgsqlBinaryImporter.Write<T>` 写入器仍计划后续实现。

`IncludeManyAsync` 的父/子 key 读取同样使用 `EntityValueAccessorRegistry`，避免 split query 组装阶段回到 `PropertyInfo.GetValue`。

生成：

```csharp
writer.Write(entity.Name, NpgsqlDbType.Text);
writer.Write(entity.Tags, NpgsqlDbType.Array | NpgsqlDbType.Text);
```

### 11.4 DbContextGenerator

生成实体注册表：

```csharp
EntityModelRegistry.Register<User>(UserEntityModel.Instance);
```

## 12. Analyzer

当前已实现基础版：Analyzer 随主包发布到 `analyzers/dotnet/cs`，用户安装 `Perigon.PostgreSQL` 后自动启用。首批诊断：

- `PG001`：检测无可见 `Where(...)` 的 `ToDeleteSql` / `ExecuteDeleteAsync` / `ToUpdateSql` / `ExecuteUpdateAsync`，提示添加过滤条件或显式 full-table options。
- `PG002`：检测 Perigon 终端 API 前查询表达式中的明确不支持方法调用，例如本地方法、`Expression.Invoke`、`System.Math` 调用、culture-aware / `StringComparison` 字符串重载。
- `PG003`：检测 `[Column(TypeName = "jsonb")]` 标注的动态 JSON POCO 属性，提示改用 `string`、`JsonDocument`、`JsonElement` 或 source-generated JSON 路径。

Analyzer 应覆盖：

- 不支持的表达式（已实现基础版：方法调用类诊断）。
- 未映射实体（计划支持）。
- 未映射属性（计划支持）。
- 动态 JSON POCO 使用（已实现基础版）。
- 可能触发反射的配置（计划支持）。
- 无 `Where` 的 update/delete（已实现基础版）。
- DbContext 并发使用（计划支持）。
- AOT 不安全 API（计划支持）。

## 13. 事务与连接

### 13.1 DataSource

优先使用 `NpgsqlDataSource`。

如果用户追求最小 AOT 体积，可支持 `NpgsqlSlimDataSourceBuilder`。

### 13.2 事务

提供：

```csharp
await db.TransactionAsync(async ct =>
{
    await db.Users.InsertAsync(user, ct);
    await db.Blogs.InsertAsync(blog, ct);
    return true;
});
```

嵌套事务策略必须明确：

- 禁止嵌套。
- 或使用 savepoint。

第一版建议禁止隐式嵌套，后续再支持 savepoint。

## 14. 日志与诊断

日志应包含：

- SQL 文本。
- 参数数量。
- 执行耗时。
- affected rows。
- 查询标签。
- 是否 split query。

默认不输出敏感参数值。需要显式开启 sensitive logging。
