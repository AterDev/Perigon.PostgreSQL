# 模型定义文档

## 概述

Perigon.PostgreSQL 使用 C# POCO 类定义数据模型。模型通过源码生成器自动发现，无需手动编写注册代码。

有两种模型定义方式：

- **约定定义（Convention）** — 按命名规则自动映射表名、列名、主键。
- **特性定义（Attribute）** — 通过 `[Table]`、`[Column]`、`[NotMapped]` 等特性精确控制映射。

---

## 基础模型（约定定义）

```csharp
public sealed class User
{
    public int Id { get; set; }              // 主键 → "id"，自增
    public string UserName { get; set; } = ""; // → "user_name"
    public int Age { get; set; }             // → "age"
    public bool IsActive { get; set; }       // → "is_active"
    public string? Status { get; set; }      // → "status"
    public DateTime CreatedAt { get; set; }  // → "created_at"
    public DateTime? UpdatedAt { get; set; } // → "updated_at"
}
```

### 约定规则

| 规则 | 示例 | 结果 |
|------|------|------|
| 表名 | `User` | `users`（类名转蛇形 + s） |
| 列名 | `UserName` | `user_name`（属性名转蛇形） |
| 主键 | `Id` 或 `{TypeName}Id` | 自动识别为主键 |
| 自增 | 主键且类型为 int/long/short | 默认 `GENERATED ALWAYS AS IDENTITY` |

## 显式模型定义（特性）

```csharp
using Perigon.PostgreSQL.Attributes;

[Table("app_users", Schema = "security")]
public sealed class AppUser
{
    [Column("user_id", IsPrimaryKey = true, IsIdentity = true)]
    public int Key { get; set; }

    [Column("display_name")]
    public string Name { get; set; } = "";

    [Column("email")]
    public string Email { get; set; } = "";

    [Column("score", IsGenerated = true)]
    public int? ComputedScore { get; set; }

    [NotMapped]
    public string RuntimeOnly { get; set; } = "";
}
```

### `[Table]`

| 参数 | 说明 |
|------|------|
| 第一个参数 | 表名，不指定则使用约定名称 |
| `Schema` | 数据库 schema，默认 `public` |

### `[Column]`

| 参数 | 说明 |
|------|------|
| 第一个参数 | 列名，不指定则使用约定蛇形名称 |
| `IsPrimaryKey` | 标记为主键 |
| `IsIdentity` | 标记为自增列 |
| `IsGenerated` | 标记为数据库生成列（不在 INSERT 中发送值） |
| `IsArray` | 标记为数组列。**通常不需要**——参见下文"数组与列表" |
| `TypeName` | 数据库类型名，例如 `jsonb` |

### `[NotMapped]`

标记该属性不参与数据库映射。

## 数组与列表

`int[]`、`string[]`、`List<string>`、`List<int>` 等类型**默认自动映射**为 PostgreSQL 数组列，无需额外特性。

```csharp
public sealed class Article
{
    public int Id { get; set; }

    public string Title { get; set; } = "";

    // 自动映射为 text[]（PostgreSQL 字符串数组）
    public string[] Tags { get; set; } = [];

    // 自动映射为 integer[]
    public List<int> Scores { get; set; } = [];

    // 可为 null 的数组
    public int[]? OptionalIds { get; set; }
}
```

支持的数组类型：

| C# 类型 | PostgreSQL 类型 |
|---------|----------------|
| `int[]` | `integer[]` |
| `long[]` | `bigint[]` |
| `short[]` | `smallint[]` |
| `string[]` | `text[]` |
| `bool[]` | `boolean[]` |
| `DateTime[]` | `timestamp with time zone[]` |
| `List<T>` | 对应 `T[]` |

> 注意：`List<T>` 属性需要 `=` 初始化或在构造函数中初始化，否则查询返回时可能为 null。推荐使用 `= [];` 默认值。

## JSONB 列

通过 `[Column(TypeName = "jsonb")]` 标记 JSONB 列。C# 侧通常使用 `string` 类型。

```csharp
using Perigon.PostgreSQL.Attributes;

public sealed class Product
{
    public int Id { get; set; }

    public string Name { get; set; } = "";

    [Column(TypeName = "jsonb")]
    public string? Attributes { get; set; }
}
```

使用 JSONB 查询方法（`JsonbContains`、`JsonbHasKey`、`JsonbText`、`JsonbPathExists`），参见查询文档。

## 自增与生成列

### 身份列（Identity）

默认情况下，类型为 `int`/`long`/`short` 的主键属性会自动被识别为身份列，生成：
```sql
id integer GENERATED ALWAYS AS IDENTITY PRIMARY KEY
```

通过 `[Column(IsIdentity = false)]` 可禁用此行为。

### 生成列（Computed / Generated）

通过 `[Column(IsGenerated = true)]` 标记数据库端生成的值列（如默认值、计算列），INSERT 时不会发送该列的值。

```csharp
[Column("slug", IsGenerated = true)]
public string? Slug { get; set; }
```

---

## 可空性

| C# 类型 | 数据库类型 | NULL 约束 |
|---------|-----------|-----------|
| `int` | `integer` | `NOT NULL` |
| `int?` | `integer` | 无 NOT NULL（即可空） |
| `string` | `text` | `NOT NULL` |
| `string?` | `text` | 可空 |
| `int[]` | `integer[]` | `NOT NULL` |
| `int[]?` | `integer[]` | 可空 |

---

## 复合主键

通过 `[Column(IsPrimaryKey = true)]` 标记多个列来定义复合主键：

```csharp
public sealed class OrderItem
{
    [Column(IsPrimaryKey = true)]
    public int OrderId { get; set; }

    [Column(IsPrimaryKey = true)]
    public int ItemIndex { get; set; }

    public string ProductName { get; set; } = "";

    public int Quantity { get; set; }
}
```

## 完整示例

```csharp
using Perigon.PostgreSQL.Attributes;

[Table("blog_posts", Schema = "content")]
public sealed class BlogPost
{
    public int Id { get; set; }

    public string Title { get; set; } = "";

    [Column("body_md")]
    public string Markdown { get; set; } = "";

    [Column(TypeName = "jsonb")]
    public string? Metadata { get; set; }

    public string[] Tags { get; set; } = [];

    public List<string> Contributors { get; set; } = [];

    [Column("is_published")]
    public bool? Published { get; set; }

    [Column(IsGenerated = true)]
    public DateTime? PublishedAt { get; set; }

    public DateTime CreatedAt { get; set; }

    [NotMapped]
    public string TempDraft { get; set; } = "";
}
```

生成的表结构：

```sql
CREATE TABLE content.blog_posts (
    id integer GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    title text NOT NULL,
    body_md text NOT NULL,
    metadata jsonb,
    tags text[] NOT NULL,
    contributors text[] NOT NULL,
    is_published boolean,
    published_at timestamp with time zone,
    created_at timestamp with time zone NOT NULL
);
```

## 在 DbContext 中注册

模型在 `DbContext` 子类中通过 `DbSet<T>` 属性暴露：

```csharp
using Perigon.PostgreSQL;

public sealed class AppDbContext : DbContext
{
    public AppDbContext(string connectionString)
        : base(builder => builder.UsePostgres(connectionString))
    {
    }

    public DbSet<User> Users => Set<User>();
    public DbSet<BlogPost> Posts => Set<BlogPost>();
    public DbSet<OrderItem> OrderItems => Set<OrderItem>();
}
```

所有 `DbSet<T>` 属性中的实体类型会被源码生成器自动发现并生成注册代码。
